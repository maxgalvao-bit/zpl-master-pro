## Ciclo Realizado: ZplEngine (Fase 4 - Início)
*   **Criação do ZplEngine (`src/services/ZplEngine.ts`):** O Motor principal agnóstico foi codificado. O parser `zpl-renderer-js` nativo do client foi injetado (via promise await `zplToBase64MultipleAsync`), tipado severamente a partir do JSON Schema estabelecido e retornando os PNGs englobando suporte nativo a etiquetas múltiplas e Data URI formatado.
*   **Editor UI e Drag&Drop (`src/components/ZplEditor.tsx`):** A UI do motor foi criada (substituindo o antigo navegador na Homepage). A interface 100% Client-Side é dividida em duas colunas (G-Max style). A API FileReader nativa do React foi embutida no Input via eventos de estado, permitindo colar código-texto ou largar aquivos `.txt`/`.zpl` na textarea e renderizar base64 nativo na lateral.
*   **Skill Customizada:** A skill de `i18n-localization` foi desmembrada do Github e instalada via linha de comando seguindo a arquitetura em 3 camadas (O `SKILL.md` foi convertido em `directives/02_i18n_localization.md` e o script validador alocado em `tools/i18n_checker.py`).

## Ciclo Realizado: Action Bar e Motor PDF (ExportEngine)
*   **Serviço Agnóstico (ExportEngine):** Classes estáticas construídas usando o `jspdf` via `window.print` (Termal) e download instantâneo (PDF A4 - Calculado Grid 2x2 centrado - e PDF 4x6 unitário). Tudo estritamente contido no navegador para proteção de dados do cliente G-Max.
*   **UI Adicionada:** Interface Action Bar instalada com contador dinâmico de labels rastreado no payload, embutida no rodápé de preview do ZplEditor. Strings foram atreladas e atualizadas simultaneamente por todo o array JSON da invariante i18n (`pt-br`, `es`, `en`, `zh`).
*   **Ajustes Críticos de UX (PDF e Preview):** Escalou-se a matemática do grid A4 para 95x142mm evitando cortes na sangria física da impressora de destino. A UI de print Térmico foi equipada com self-destroy no `onafterprint()`. O ZplEditor recebeu Loading States robustos e bloqueio preventivo de múltiplos gatilhos simultâneos. Botões ganharam peso G-Max (height/padding aumentados e SVGs nativos).
*   **Correção de Layout (Action Bar):** Os botões que estavam preenchendo erraticamente a tela foram contidos em um Grid Responsivo CSS (`grid-cols-3`). Os ícones SVG internos também receberam classes puristas rigorosas (`w-6 h-6 flex-shrink-0 mr-2`) para impedi-los de vazar e corromper o painel de ZPL View.
